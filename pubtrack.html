PubTrack

Description
PubTrack is part of the PubSearch web-based literature curation toolset and its primary goal is to function as a tracking mechanism for literature articles as they pass through the curation process. Each MOD's curation procedures are likely to be different so PubTrack provides the ability to store multiple customizable procedures designed by the user, allowing (in theory) any uniquely identifiable object to be tracked through a fairly complex set of procedures. A standard API will be provided for other scripts/modules to add and delete objects from the tracking database and to query and update an object's status, this will be provided using a webservice model. We will develop this as a tool for tracking literature objects (papers) through the curation process though we hope it will find utility in other areas of MOD operations.

Demo & Screenshots

Demos and screenshots are available at http://castor.brc.mcw.edu/pubtrack

Requirements

The module will be probably be prototyped in perl, we will very likely migrate to java servlets for future revisions to provide closer integration with PubSearch. The database will be prototyped in MySQL and ported to PostGres and Oracle. Perl DBI will be used for perl database access, JDBC will be used for the java-based version.
