Notes: 2003-07-14 The following represents the documentation of currently active tables. Inconsistencies are minor but  there are a few major clean-up that is required:

1. Separating out TAIR and other External DB ids into a generic XDB_REF

2. Moving Replaced_by to UpdateHistory

3. Separation of NucleotideSequence and AASequence from pub_sequence table.

 term

 The terms table contains all terms that can potentially be used for indexing the articles. There are 12 types of terms including the 3 GO types process (proc), function (func), compartment (comp) plus the root type, which is above the three GO aspects. The anatomy (anat) and development (deve) types are vocabularies defined at TAIR, user defined terms (user) are essentially terms that user use to describe microarray experiments in the TAIR database. The 'gene' type denotes the terms that are actual or potential gene names. If a term is a gene term, it is linked to a pub_gene entry.

21

pub_locus

This table contains locus information. It is linked to pub_gene through a linking table, so that many genes can be linked to a locus, and conversely also many loci to one gene. This allows to represent trans-splicing in organellar genomes.  

22

pub_gene_locus

This table is a linking table between pub_gene and pub_locus. The reason for having a many to many relation ship between genes and loci are the trans-spliced genes in the mitochondrial genome.

23

term_annotation

This table stores the annotations; this is essentially a linking table between two terms. The subject term is usually the gene to be annotated and the object term is the controlled vocabulary term. The table also contains foreign keys to the evidence, reference, and relationship_type and other information relating to annotations.

24

evidence_type

This table stores different types of evidence used for annotations stored in term_annotation table. The types are consistent with what is being used by GO consortium.
