 Table Name Description
1 term The terms table contains all terms that can potentially be used for indexing the articles. There are 12 types of terms including the 3 GO types process (proc), function (func), compartment (comp) plus the root type, which is above the three GO aspects. The anatomy (anat) and development (deve) types are vocabularies defined at TAIR, user defined terms (user) are essentially terms that user use to describe microarray experiments in the TAIR database. The 'gene' type denotes the terms that are actual or potential gene names. If a term is a gene term, it is linked to a pub_gene entry.

hit

This table stores information about matches between terms and articles. Currently the ‘hits’ are generated by exact matching using suffix trees. We are currently testing using Lucene algorithm as well as using fgrep utility to improve performance. The entries in this table can be validated manually by curators.

Field name

 Datatype Definition
match_id int(12) the primary key of this table.
term_id int(12) foreign key to the term table (term_id). The term that was used to generate this hit.
match_nr int(11) How many times the term matches in this article.
article_id int(12) foreign key to the article table (article_id). The article this hit is found in.
in_title enum('y','n') Denotes whether the hit occurred in the title.
in_abstract enum('y','n') Denotes whether the hit occurred in the abstract.
is_valid enum('y','n','m') whether the hit is valid. This field is usually set manually by a curator who inspects hits for a specific gene. Contains NULL if the hit has not been verified, 'y' if valid, 'n' if not valid and 'm' if the hit may be valid.
validated_by int(12) foreign key to the user table. The user who validated this hit.

entered_by int(12) foreign key to user table (user_id) for the user that entered the data.
Is_obsolete enum('y','n') whether the hit is obsolete. Hits are obsoleted if the associated term or article is obsoleted.         
date_updated date The date the record has been last updated.               
updated_by int(12) foreign key to the user table. The user who last updated the hit.            
date_last_synchronized date the date the last synchronization occurred with the TAIR database.

The pub_gene table contains all the information specific to a TAIR gene model. A TAIR gene model essentially captures the transcript and protein information and is linked to a locus, which denotes a segment on the chromosome.

is_verified enum('Y','N') Describes whether the gene structure of this gene has been verified experimentally but is populated inconsistently in the database.
